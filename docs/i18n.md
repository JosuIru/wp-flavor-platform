# Internacionalización (i18n) de Visual Builder Pro

Este documento describe el sistema de internacionalización del Visual Builder Pro y cómo traducir el plugin a otros idiomas.

## Descripción General

Visual Builder Pro utiliza un sistema de internacionalización completo que incluye:

- **PHP**: Funciones estándar de WordPress (`__()`, `_e()`, `esc_html__()`, etc.)
- **JavaScript**: Sistema personalizado `VBPi18n` que se integra con las traducciones de WordPress
- **Archivos de traducción**: Formato PO/MO estándar de WordPress

## Estructura de Archivos

```
flavor-platform/
├── languages/
│   ├── flavor-platform.pot          # Template de traducción (fuente)
│   ├── flavor-platform-es_ES.po     # Traducciones en español
│   ├── flavor-platform-es_ES.mo     # Archivo compilado (español)
│   ├── flavor-platform-en_US.po     # Traducciones en inglés
│   └── flavor-platform-en_US.mo     # Archivo compilado (inglés)
├── assets/vbp/js/
│   └── vbp-i18n.js                  # Sistema i18n para JavaScript
└── includes/visual-builder-pro/
    └── class-vbp-editor.php         # Función obtener_strings_i18n()
```

## Text Domain

El text domain del plugin es: `flavor-platform`

Todas las funciones de traducción deben usar esta constante:

```php
__( 'Texto a traducir', 'flavor-platform' );
// O usar la constante definida:
__( 'Texto a traducir', FLAVOR_PLATFORM_TEXT_DOMAIN );
```

## Sistema i18n en PHP

### Funciones Disponibles

| Función | Uso | Ejemplo |
|---------|-----|---------|
| `__()` | Traducir y retornar | `$texto = __( 'Guardar', 'flavor-platform' );` |
| `_e()` | Traducir y hacer echo | `_e( 'Guardar', 'flavor-platform' );` |
| `esc_html__()` | Traducir y escapar HTML | `esc_html__( 'Texto', 'flavor-platform' )` |
| `esc_attr__()` | Traducir y escapar atributo | `esc_attr__( 'Título', 'flavor-platform' )` |
| `esc_html_e()` | Traducir, escapar y echo | `esc_html_e( 'Texto', 'flavor-platform' )` |
| `_n()` | Plurales | `_n( '1 elemento', '%d elementos', $count, 'flavor-platform' )` |
| `_x()` | Contexto | `_x( 'Post', 'noun', 'flavor-platform' )` |
| `_nx()` | Plural con contexto | `_nx( '1 item', '%d items', $n, 'menu', 'flavor-platform' )` |

### Agregar Nuevos Strings en PHP

1. Localiza el archivo PHP donde necesitas agregar el string
2. Usa la función apropiada de traducción
3. Ejecuta el script de extracción para actualizar el POT

```php
// Ejemplo: Agregar nuevo mensaje de error
$mensaje = __( 'No se pudo guardar el documento', 'flavor-platform' );
```

## Sistema i18n en JavaScript

### Funciones Disponibles

El archivo `vbp-i18n.js` proporciona las siguientes funciones globales:

```javascript
// Traducción simple
__('save', 'Guardar')  // Retorna traducción o fallback

// Traducción con sustitución (sprintf-style)
_s('elements_selected', 5)  // "5 elementos seleccionados"

// Plurales
_n('1 elemento', '%d elementos', count)

// Traducción con contexto
_x('Post', 'noun', 'Publicación')
```

### Agregar Nuevos Strings en JavaScript

1. Primero, añade el string en PHP en `class-vbp-editor.php`:

```php
// En el método obtener_strings_i18n()
'myNewString' => __( 'Mi nuevo texto', 'flavor-platform' ),
```

2. Luego usa el string en JavaScript:

```javascript
// El string se carga automáticamente
var texto = __('myNewString', 'Mi nuevo texto por defecto');
```

### Convención de Nombres de Keys

Usa nombres descriptivos en camelCase:

- `save` - Acciones simples
- `deleteConfirm` - Confirmaciones
- `errorNetwork` - Errores
- `blockHeading` - Tipos de bloques
- `categoryLayout` - Categorías

## Cómo Traducir el Plugin

### Para Traductores

#### Opción 1: Usando Poedit

1. Instala [Poedit](https://poedit.net/) (gratuito)
2. Abre el archivo `languages/flavor-platform.pot`
3. Crea una nueva traducción desde el catálogo
4. Traduce cada string
5. Guarda como `flavor-platform-{locale}.po` (ej: `flavor-platform-fr_FR.po`)
6. Poedit genera automáticamente el archivo `.mo`

#### Opción 2: Usando translate.wordpress.org

Si el plugin está en el repositorio de WordPress.org:

1. Ve a [translate.wordpress.org](https://translate.wordpress.org)
2. Busca "Flavor Platform"
3. Selecciona tu idioma
4. Traduce los strings pendientes

#### Opción 3: Usando Loco Translate (Plugin WordPress)

1. Instala el plugin [Loco Translate](https://wordpress.org/plugins/loco-translate/)
2. Ve a Loco Translate > Plugins
3. Selecciona Flavor Platform
4. Crea nueva traducción
5. Traduce y guarda

## Para Desarrolladores

### Extraer Strings del Código

Ejecuta el script de extracción:

```bash
cd /ruta/al/plugin

# Usando WP-CLI
wp i18n make-pot . languages/flavor-platform.pot --domain=flavor-platform

# O usando el script personalizado
bash scripts/extract-strings.sh
```

### Agregar un Nuevo Idioma

1. Copia `languages/flavor-platform.pot` a `flavor-platform-{locale}.po`
2. Edita el encabezado del archivo `.po`:

```
"Language: fr_FR\n"
"Plural-Forms: nplurals=2; plural=(n != 1);\n"
```

3. Traduce los strings
4. Compila el archivo `.mo`:

```bash
msgfmt flavor-platform-fr_FR.po -o flavor-platform-fr_FR.mo
```

### Verificar Traducciones

```bash
# Verificar sintaxis del archivo PO
msgfmt --check flavor-platform-es_ES.po

# Ver strings sin traducir
msgattrib --untranslated flavor-platform-es_ES.po
```

### Buenas Prácticas

1. **Siempre usar el text domain**: No dejar strings hardcodeados
2. **Escapar correctamente**: Usar `esc_html__()` para HTML, `esc_attr__()` para atributos
3. **Evitar concatenación**: Mal: `__('Hola') . ' ' . __('Mundo')`. Bien: `__('Hola Mundo')`
4. **Usar placeholders**: `sprintf(__('Hola %s'), $nombre)`
5. **Proporcionar contexto**: Usar `_x()` cuando un string pueda ser ambiguo
6. **Plurales correctos**: Usar `_n()` para cantidades

### Estructura de Strings Centralizada

Los strings de JavaScript están centralizados en `class-vbp-editor.php`:

```php
private function obtener_strings_i18n() {
    return array(
        // Acciones
        'save'    => __( 'Guardar', 'flavor-platform' ),
        'cancel'  => __( 'Cancelar', 'flavor-platform' ),

        // Estados
        'loading' => __( 'Cargando...', 'flavor-platform' ),

        // Errores
        'error'   => __( 'Ha ocurrido un error', 'flavor-platform' ),

        // ... más strings
    );
}
```

## Pruebas

### Verificar que las Traducciones Funcionan

1. Cambia el idioma de WordPress (Ajustes > General > Idioma del sitio)
2. Abre el editor Visual Builder Pro
3. Verifica que los textos aparezcan traducidos

### Modo Debug en JavaScript

Activa el modo debug para ver strings sin traducir:

```javascript
// En la consola del navegador
VBPi18n.enableDebug();
```

Los strings sin traducción aparecerán como warnings en la consola.
