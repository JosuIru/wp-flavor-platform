# Visual Builder Pro 2.0.0 - Release Notes

**Fecha de lanzamiento:** 1 de Junio de 2025
**Tipo de release:** Mayor (breaking changes)
**Compatibilidad:** WordPress 6.0+, PHP 7.4+

---

## Resumen Ejecutivo

Visual Builder Pro 2.0.0 representa una reescritura completa del editor visual, pasando de un editor basico a un sistema profesional comparable a Figma o Photoshop. Esta version introduce un editor fullscreen, sistema de simbolos, colaboracion en tiempo real, y mas de 15 nuevas funcionalidades.

---

## Highlights

### Editor Visual Fullscreen

El nuevo editor ocupa toda la pantalla del navegador, proporcionando el maximo espacio de trabajo posible.

**Layout del Editor:**
```
+---------------------------------------------------------------------+
|                        TOOLBAR SUPERIOR                              |
|  [Logo] [Undo][Redo] | [Desktop][Tablet][Mobile] | [Zoom] [Save]    |
+----------+---------------------------------------------+-------------+
|  PANEL   |                 C A N V A S                 | INSPECTOR   |
|  BLOQUES |           (Drag & Drop con Sortable)        |  ESTILOS    |
|          |                                             |             |
|  CAPAS   |  [Elementos editables inline]               |   LAYOUT    |
+----------+---------------------------------------------+-------------+
|                        STATUS BAR                                    |
+---------------------------------------------------------------------+
```

**Caracteristicas principales:**
- Toolbar con controles de deshacer/rehacer, breakpoints y zoom
- Panel izquierdo con bloques disponibles y arbol de capas
- Canvas central con drag & drop nativo
- Inspector derecho con todas las propiedades del elemento seleccionado
- Barra de estado con informacion contextual

---

### Sistema de Simbolos

Los simbolos son componentes reutilizables cuyas instancias se mantienen sincronizadas automaticamente.

**Casos de uso:**
- Headers y footers consistentes en todas las paginas
- Tarjetas de producto/servicio con el mismo diseno
- Botones y elementos de UI reutilizables
- Componentes que se actualizan globalmente

**Funcionalidades:**
- Crear simbolo desde seleccion (`Ctrl+Shift+Y`)
- Insertar instancias vinculadas (`Ctrl+Alt+O`)
- Overrides por instancia (personalizar texto, imagenes, colores)
- Variantes de simbolo (ej: boton primario, secundario, ghost)
- Simbolos anidados (simbolos dentro de simbolos)
- Import/Export como JSON

---

### Colaboracion en Tiempo Real

Multiples usuarios pueden editar el mismo documento simultaneamente.

**Tecnologia:**
- WebSocket server para sincronizacion instantanea
- Fallback a WordPress Heartbeat si WebSocket no disponible
- CRDT para resolucion de conflictos

**Experiencia de usuario:**
- Ver cursores de otros usuarios en tiempo real
- Indicador de quien esta editando cada elemento
- Locks optimistas (sin bloqueos exclusivos)
- Avatar y nombre del usuario junto a su cursor

---

### Modo Prototipado

Crear prototipos interactivos sin salir del editor.

**Interacciones soportadas:**
- Click, hover, scroll
- Navegacion entre frames
- Mostrar overlays/modales
- Animar elementos

**Exportacion:**
- Preview interactivo en navegador
- Exportar como HTML estatico
- Compartir enlace de preview

### AI Layout Assistant

La IA ayuda a crear y mejorar layouts automaticamente.

**Capacidades:**
- Generar estructura desde descripcion ("Crea un hero con imagen y CTA")
- Auto-spacing inteligente entre elementos
- Sugerencias de colores complementarios
- Generar variantes de un diseno existente

---

### Smart Guides y Spacing

Herramientas visuales para alineacion precisa.

**Smart Guides:**
- Lineas de alineacion automaticas al arrastrar
- Snap a grid de 8px
- Snap a bordes y centros de otros elementos

**Spacing Indicators:**
- Ver distancias entre elementos
- Activar con Alt+hover
- Valores en pixels claramente visibles

---

### Constraints/Pinning

Sistema de anclaje tipo Figma para layouts responsive.

**Opciones de anclaje:**
- Top, Right, Bottom, Left
- Center Horizontal, Center Vertical
- Fill (llenar espacio disponible)

**Comportamiento:**
- Los elementos mantienen su posicion relativa al redimensionar
- Presets rapidos para casos comunes
- Visualizacion de constraints en canvas

---

### Global Styles

Estilos CSS centralizados y reutilizables.

**Gestion:**
- Definir clases como `.heading-1`, `.button-primary`
- Aplicar desde dropdown en inspector
- Editar actualiza todas las instancias

**Exportacion:**
- Exportar como archivo CSS
- Variables CSS incluidas

---

### Animation Builder

Crear animaciones CSS visualmente.

**Timeline:**
- Arrastrar keyframes en linea de tiempo
- Curvas de easing predefinidas y custom
- Preview en canvas

**Triggers:**
- Al cargar la pagina
- Al hacer scroll (scroll-into-view)
- Al hover
- Al click

**Exportacion:**
- Genera CSS @keyframes valido
- Incluir en el frontend automaticamente

---

### Asset Manager

Biblioteca centralizada de medios.

**Caracteristicas:**
- Panel lateral con todos los assets
- Tabs: Imagenes, SVGs, Videos, Iconos
- Drag & drop para subir
- Busqueda y filtros
- Integracion con Unsplash

---

### Sistema de Branching

Ramas de diseno para experimentar sin afectar el original.

**Funcionalidades:**
- Crear branch desde cualquier punto
- Nombrar y describir branches
- Merge con resolucion de conflictos
- Diff visual entre branches

**Implementacion:**

## Breaking Changes

### Migrado de jQuery UI a SortableJS

**Antes (1.x):**
```javascript
$('#canvas').sortable({
    items: '.vbp-element',
    // ...
});
```

**Ahora (2.0):**
```javascript
new Sortable(canvas, {
    draggable: '.vbp-element',
    // ...
});
```

**Accion requerida:** Si tenias extensiones que dependian de jQuery UI sortable, migrar a la API de SortableJS.

---

### Nuevo sistema de estados con Alpine.js

**Antes (1.x):**
```javascript
window.VBP = {
    selectedElement: null,
    // ...
};
```

**Ahora (2.0):**
```javascript
Alpine.store('vbp', {
    selectedElements: [],
    // ...
});
```

**Accion requerida:** Acceder al estado via `Alpine.store('vbp')`.

---

### Estructura de datos de paginas

El formato de almacenamiento ha cambiado. Las paginas creadas con 1.x se migraran automaticamente al abrir en 2.0.

**Antes (1.x):**
```json
{
    "elements": [...]
}
```

**Ahora (2.0):**
```json
{
    "version": "2.0",
    "elements": [...],
    "symbols": [...],
    "globalStyles": [...],
    "metadata": {...}
}
```

---

### Cambios en API REST

| Endpoint 1.x | Endpoint 2.0 | Notas |
|--------------|--------------|-------|
| `/vbp/v1/page` | `/flavor-vbp/v1/pages` | Pluralizado |
| `/vbp/v1/save` | `/flavor-vbp/v1/pages/{id}` (PUT) | RESTful |
| `/vbp/v1/blocks` | `/flavor-vbp/v1/claude/blocks` | Namespace claude |

## Guia de Migracion

### Paso 1: Backup

```bash
# Backup de base de datos
wp db export backup-pre-vbp2.sql

# Backup de archivos
zip -r backup-wp-content.zip wp-content/
```

### Paso 2: Actualizar Plugin

```bash
# Via WP-CLI
wp plugin update flavor-platform

# O subir ZIP manualmente desde admin
```

### Paso 3: Ejecutar Migracion de Datos

La migracion se ejecuta automaticamente al activar. Para forzar:

```bash
wp eval "do_action('flavor_vbp_migrate_data');"
```

### Paso 4: Verificar Paginas

1. Abrir cada pagina VBP en el editor
2. Verificar que el contenido se renderiza correctamente
3. Guardar para confirmar migracion

### Paso 5: Actualizar Extensiones

Si tienes codigo personalizado que interactua con VBP:

1. Migrar de jQuery UI a SortableJS
2. Acceder estado via Alpine.store
3. Actualizar llamadas a API REST

## Deprecaciones

Las siguientes funcionalidades estan obsoletas y se eliminaran en 3.0:

| Obsoleto | Reemplazo |
|----------|-----------|
| `VBP.getState()` | `Alpine.store('vbp')` |
| `$('#vbp-canvas').sortable()` | `new Sortable()` |
| Editor modal | Editor fullscreen |
| Panel de widgets | Panel de bloques |

## Problemas Conocidos

1. **Firefox < 90**: Algunos efectos de drag pueden parpadear
2. **Safari iOS**: Zoom con pinch requiere doble tap inicial
3. **IE11**: No soportado (usar Edge)
